# Why Blog?

저는 글을 쓰고 제 생각을 적는 것을 좋아하는 편인데

또 너무 무거운 주제로 멋지게 써내려가는 글을 내가 적는 것은 오글거려하는 그런 성격입니다.

나중에 제가 작성한 글을 읽어보면 부족함이 너무 잘 보이거든요

그럼에도 여러 사람들이 제 글을 읽어준다는 건 꽤 기분이 좋은 일입니다.

그래서 글을 쓰는 것을 멈출 수 없는 것 같습니다.

티스토리에서 이미 기술 블로그를 열심히 운영하고 있고 [티스토리 블로그](https://xionwcfm.tistory.com/)

누적 글도 현재 300개가 넘어가고 방문자 수도 나쁘지않게 나오는 편이지만

불편한 점이 몇가지 있었습니다.

그리고 공을 많이 들인 포스트가 다른 포스트들에 묻히는 것도 마음이 좀 아팠어요

## 티스토리는...

티스토리는 대체로 좋았습니다. 네이버 블로그에 비해 자유도 높은 스킨도 그렇고

서치콘솔만 잘 등록해두면 구글에 검색이 잘되는 점도 좋았습니다.

하지만 아쉬운 점도 당연히 존재했는데 스킨을 만들거나 수정하는 작업이

굉장히 지저분하고 따라하기 싫게 되어 있었습니다.

자연스레 남이 만들어둔 스킨을 이용하게 되었는데 그러다보니

미세하게 마음에 들지 않는 부분들을 참고 쓰게 되었습니다.

내가 원하는 기능들이 충분히 들어간 블로그를 만들 수 있을 것 같다는 생각이 들었고

재미있을 것 같아 개인 블로그를 만드는 프로젝트를 시작했습니다.

## 어떻게 만드는건데?

블로그를 만드는 여러가지 방법이 존재합니다.

`Gatsby` , `jekyll` 등 많은 정적 사이트 생성기들이 있어요!

물론 각자 조금씩 장단도 존재합니다. 개츠비의 경우에는 풍부한 라이브러리 생태계가 존재하고

리액트 기반으로 돌아간다는 장점이 있지만 `GraphQL`에 의존성이 있다는 점이 좀 걸렸습니다.

`GraphQL`을 선호하시는 분들은 상관없을 수 있지만요!

결론적으로 이야기하면 Next.js를 선택했습니다.

블로그는 결국 SEO가 중요한 플랫폼이기 때문에 검색엔진 최적화가 중요한 데 비해

SPA로 구성한 프로젝트는 이 부분이 큰 약점이 되기 때문입니다

Next.js는 SSG도 지원해주고 이미 Next.js를 이용해 블로그를 구성한 선례들도 쉽게 찾아볼 수 있어

더욱 쉽게 시작해볼 수 있었습니다.

## 배포는 어떻게?

처음 배포 서비스를 선정할 때 가장 크게 생각한 기준은 비용이었습니다.

혹시라도 내 블로그가 잘되어서 트래픽이 엄청나게 터질수도 있으니까..

라는 생각으로 Cloud Flare를 이용하려고 했습니다.

무료플랜도 정적호스팅은 무제한으로 가능한 플랫폼이었거든요

다만 SSR을 위해서는 약간의 추가 설정이 더 필요했는데

Cloud Flare가 제공하는 Worker를 사용하기 위한 설정에서 조금 문제가 생겼습니다.

따라서 배포 사이트는 vercel로 옮기게 되었습니다.

## 어려웠던 건 없어?

사실 그렇게 까다로운 로직이 들어가진 않았습니다.

오히려 시간이 많이 드는 부분은 코드 작성이 아니라 포스팅을 작성하기 위한 설정들이었습니다.

예컨대 MDX로 작성된 파일을 적절히 렌더시켜주기 위한 방법

혹은 유려한 화면 전환을 위한 CSS 애니메이션

반응형으로 사이트를 구성하기

미약하게나마 디자인 시스템을 만들기 등등에 시간이 들어가게 되었고

사실 콘텐츠들은 로컬에서 가져오게되기 때문에 데이터 페칭이 필요하지도 않고

상태를 관리해줄 필요도 없으니 까다로운 건 CSS 정도인 것 같습니다.

반대로 말하면 HTML, CSS에 대한 기초적인 지식만 있다면

쉽게 개인 블로그를 만들 수 있을 것 같아요
